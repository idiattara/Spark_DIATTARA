En Spark, lors de l'criture des donnes vers un systme de stockage (comme HDFS, S3, etc.), vous pouvez spcifier diffrents modes d'criture pour grer le comportement lorsque le rpertoire de destination existe dj. Voici quelques modes courants :

Append (Ajouter) : Les donnes sont ajoutes  la fin du rpertoire existant.
Overwrite (craser) : Le rpertoire de destination est entirement remplac par les nouvelles donnes. Cela efface compltement les donnes existantes.
Ignore (Ignorer) : Si le rpertoire de destination existe dj, les donnes ne sont pas crites et aucune erreur n'est leve. Cela permet d'viter d'crire accidentellement sur des donnes existantes.
ErrorIfExists (Erreur si existe) : Une erreur est leve si le rpertoire de destination existe dj. C'est le comportement par dfaut si aucun mode n'est spcifi.
OverwritePartitions (craser les partitions) : Cette option est spcifique  Spark SQL. Elle permet d'crire uniquement les partitions qui correspondent aux donnes fournies, sans affecter les autres partitions existantes.
