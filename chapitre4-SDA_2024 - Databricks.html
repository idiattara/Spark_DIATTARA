En Spark, lors de l'écriture des données vers un système de stockage (comme HDFS, S3, etc.), vous pouvez spécifier différents modes d'écriture pour gérer le comportement lorsque le répertoire de destination existe déjà. Voici quelques modes courants :

Append (Ajouter) : Les données sont ajoutées à la fin du répertoire existant.
Overwrite (Écraser) : Le répertoire de destination est entièrement remplacé par les nouvelles données. Cela efface complètement les données existantes.
Ignore (Ignorer) : Si le répertoire de destination existe déjà, les données ne sont pas écrites et aucune erreur n'est levée. Cela permet d'éviter d'écrire accidentellement sur des données existantes.
ErrorIfExists (Erreur si existe) : Une erreur est levée si le répertoire de destination existe déjà. C'est le comportement par défaut si aucun mode n'est spécifié.
OverwritePartitions (Écraser les partitions) : Cette option est spécifique à Spark SQL. Elle permet d'écrire uniquement les partitions qui correspondent aux données fournies, sans affecter les autres partitions existantes.
